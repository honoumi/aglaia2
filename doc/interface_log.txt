数据模型(共四个类，仅成员变量Target的类型不同)：
LogAccount
LogBorrow
LogSingle
LogComputing

user_id： int，操作发出者的ID
target:   ForeignKey，被操作对象，四个类中分别为：Account,Borrow,Single,Computing
action:   String，长度上限100，动作类型，包括create、update、delete等
time：    DateTime，操作时间
description：  动作描述，如：提出借用申请



============================


前端接口：

【1】
show_log页面： //显示特定对象的相关日志条目
GET方法，
	type：日志类型（goods | computing | user | borrow）
	id:   日志查询对象的id
	is_actor:  如果查询用户日志，此项表示筛选选项（ true：该用户的操作日志|
							false：该用户的信息修改日志 |
							All：所有和该用户相关的日志）
页面context:
logs:[   //所有日志条目

	{
		actor : '...'
		target : '...'
		action : '...'
		time : '...'
		desc : '...'
	},
	{
	}
	...
]
